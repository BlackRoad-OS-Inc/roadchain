Layer-by-Layer

  • Application Layer High-level services that consume smart contract data. REST APIs, webhooks, real-time event streams, and admin dashboards.
  • Smart Contract Layer Solidity 0.8.x contracts with OpenZeppelin security patterns. Upgradeable proxies for zero-downtime contract updates. Multi-signature governance for critical operations.
  • Consensus Layer Proof of Authority (Clique) consensus. Known validators only. 2-second block time with sub-100ms transaction finality. No mining, no gas fees, no speculation.
  • Network Layer Standard JSON-RPC on :8545. WebSocket subscriptions on :8546. Nodes communicate over WireGuard mesh or Tor hidden services for maximum privacy.
  • Storage Layer LevelDB for chain state, PostgreSQL for indexed queries, IPFS for document storage, MinIO for binary attachments. Every layer is self-hosted.
ConsentManager.sol 
// SPDX-License-Identifier: PROPRIETARY
pragma solidity ^0.8.19;

import "@openzeppelin/contracts/access/AccessControl.sol";
import "@openzeppelin/contracts/security/ReentrancyGuard.sol";

contract ConsentManager is AccessControl, ReentrancyGuard {
    bytes32 public constant AUDITOR_ROLE = keccak256("AUDITOR_ROLE");

    struct ConsentRecord {
        address subject;
        address controller;
        bytes32 purposeHash;
        uint256 grantedAt;
        uint256 expiresAt;
        bool    revoked;
    }

    mapping(bytes32 => ConsentRecord) public consents;
    mapping(address => bytes32[]) public subjectConsents;

    event ConsentGranted(bytes32 indexed consentId, address indexed subject, bytes32 purposeHash);
    event ConsentRevoked(bytes32 indexed consentId, address indexed subject, uint256 revokedAt);

    function grantConsent(
        address controller,
        bytes32 purposeHash,
        uint256 duration
    ) external nonReentrant returns (bytes32) {
        bytes32 consentId = keccak256(abi.encodePacked(
            msg.sender, controller, purposeHash, block.timestamp
        ));

        consents[consentId] = ConsentRecord({
            subject: msg.sender,
            controller: controller,
            purposeHash: purposeHash,
            grantedAt: block.timestamp,
            expiresAt: block.timestamp + duration,
            revoked: false
        });

        subjectConsents[msg.sender].push(consentId);
        emit ConsentGranted(consentId, msg.sender, purposeHash);
        return consentId;
    }

    function revokeConsent(bytes32 consentId) external {
        require(consents[consentId].subject == msg.sender, "Not subject");
        consents[consentId].revoked = true;
        emit ConsentRevoked(consentId, msg.sender, block.timestamp);
    }

    function isConsentValid(bytes32 consentId) external view returns (bool) {
        ConsentRecord storage c = consents[consentId];
        return !c.revoked && block.timestamp <= c.expiresAt;
    }
}

Consent Infrastructure

Consent That Actually Works

Cookie banners are theater. RoadChain consent is cryptographically verifiable, user-controlled, and legally admissible.

01

User Grants Consent

The data subject signs a consent transaction specifying the data controller, processing purpose, and duration. The transaction is mined into a block with a verifiable timestamp.

02

Controller Verifies

Before processing data, the controller queries the ConsentManager contract. The contract returns a boolean — valid or not. No ambiguity, no interpretation required.

03

Subject Revokes Anytime

The data subject can revoke consent with a single transaction. Revocation is instant, immutable, and emits an event that all listening controllers receive in real time.

Developer Interface

RoadChain REST Endpoints

In addition to standard JSON-RPC, RoadChain exposes a REST interface for common verification operations.

POST /api/v1/consent/grant
Grant consent from a data subject to a controller for a specific purpose and duration.
POST /api/v1/consent/revoke
Revoke a previously granted consent. Immediate effect, all controllers notified.
GET /api/v1/consent/:id/status
Check if a consent record is currently valid. Returns boolean with metadata.
POST /api/v1/audit/log
Append an audit entry to the immutable trail. Returns block number and tx hash.
GET /api/v1/audit/:entity/trail
Retrieve the complete audit trail for an entity. Paginated, with Merkle proof per entry.
POST /api/v1/zkp/verify
Submit a zero-knowledge proof for on-chain verification. Supports Groth16 and PLONK.
POST /api/v1/document/register
Register a document hash on-chain. Optionally include IPFS CID for decentralized storage.
GET /api/v1/document/:hash/verify
Verify a document hash exists on-chain. Returns registration timestamp and registrant.
GET /api/v1/identity/:did/claims
Retrieve verifiable claims for a DID. Returns claim types without underlying data.
POST /api/v1/compliance/check
Run a compliance check against configured rules. Returns pass/fail with detailed reasoning.
